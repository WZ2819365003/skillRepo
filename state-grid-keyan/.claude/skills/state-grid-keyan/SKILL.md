# 国家电网/南方电网科技项目可行性研究报告生成技能 v2.0

本技能用于生成符合国家电网公司（含各省/市电力公司）和南方电网公司科技项目可行性研究报告规范的完整文档。

## 核心架构：Markdown 中间件 + 分章节深度生成

```
用户输入项目主题
    ↓
[本 skill] 分 4 轮生成完整 Markdown（每轮 2-3 章，确保深度）
    ↓
合并为完整的 .md 文件
    ↓
[docx-cn skill] 将 Markdown 转换为格式规范的 .docx 文件
    ↓
输出成品 Word 文档
```

**为什么分轮生成：** 可研报告总篇幅 12000-20000 字，单次生成容易导致后面章节内容空洞。分 4 轮生成，每轮专注 2-3 章，确保每章都有足够的深度和专业性。

#### 第一章：目的和意义（目标 3000-4000 字）

本章是报告的"灵魂"，必须充分论证项目的必要性和价值。

**## 1.1 研究背景（1200-1800 字）**

必须逐层展开以下五个维度，每个维度至少 200 字：

1. **国家战略与政策维度**（≥300字）
   - "双碳"战略目标对电力行业的影响
   - 新型电力系统建设的国家要求
   - 引用具体政策文件名称和文号（如《关于加快建设新型电力系统的指导意见》《"十四五"现代能源体系规划》等）
   - 国家对科技创新的支持政策

2. **行业发展趋势维度**（≥250字）
   - 电力行业数字化转型趋势
   - 新技术（AI、大数据、物联网、云计算等）在电力领域的应用趋势
   - 国内外同行业技术发展对比

3. **电网公司战略需求维度**（≥250字）
   - 国家电网/南方电网的战略规划和科技创新方向
   - 公司"十四五"发展规划中的相关要求
   - 引用公司内部具体文件（如营销部、运检部等的通知、指引等）
   - 公司数字化转型和智能化升级的具体部署

4. **现实问题与痛点维度**（≥250字）
   - 当前电网运营/管理中存在的具体技术瓶颈
   - 现有技术方案的局限性和不足
   - 用具体数据或案例说明问题的严重性
   - 分条列出 3-5 个核心痛点

5. **项目必要性总结**（≥150字）
   - 综合上述四个维度，论证本项目提出的必要性和紧迫性
   - 点明项目的技术创新方向

**## 1.2 项目目标与意义（800-1200 字）**

结构要求：

- **项目总体目标**（1段）：一句话概括 + 2-3句话展开
- **具体研究目标**（分条）：列出 4-6 个可量化、可验证的目标，每条 1-2 句话
- **理论意义**（1段，≥200字）：对学科理论体系的贡献，与现有理论的关系
- **实践意义**（1段，≥200字）：对电网运营实际的改善，用具体指标说明
- **经济意义**（1段，≥150字）：可量化的经济效益预期

**## 1.3 成果应用和推广途径（400-600 字）**

- 应用主体和首发应用范围
- 试点计划（地区、时间、规模）
- 推广路径：本单位 → 省公司 → 国网/南网系统 → 全行业
- 推广方式：技术推广会、培训、标准化、论文专利、产学研合作

**## 1.4 成果推广后的直接和间接效益（400-600 字）**

- **直接效益**：分 4-5 条，每条 2-3 句话，尽量量化
- **间接效益**：分 4-5 条，每条 2-3 句话，从管理、人才、标准、行业影响等维度

### 第二轮：第二章（国内外研究水平综述）

#### 第二章：国内外研究水平综述（目标 5000-8000 字）

本章是学术深度和文献功底的核心体现，必须具备充分的文献支撑和深入的专业分析。

**⚠️ 核心要求：**
1. **自然段写作**：每个小节用 7-9 个自然段展开，每段 150-300 字，以学术论文综述的笔法行文，禁止使用加粗前缀+列表的AI风格
2. **真实文献**：所有参考文献必须通过 WebSearch 联网检索验证，严禁凭记忆编造。检索渠道包括 Google Scholar、IEEE Xplore、CNKI、万方等
3. **文献数量**：全文参考文献总量 60-100 条，第二章引用其中大部分
4. **文献覆盖面**：涵盖信号处理、传统机器学习、深度学习（CNN/RNN/LSTM）、图神经网络（GNN/GAT）、Transformer、GAN数据增强、强化学习自愈控制、联邦学习、模型轻量化/边缘部署等方向

**## 2.1 相关技术发展历史回顾（1500-2500 字，7-9 自然段）**

用 7-9 个自然段完整叙述技术发展的演进脉络，覆盖以下 4 个阶段：
- 第一阶段：传统继电保护方法（20世纪50年代至80年代），1-2 段
- 第二阶段：基于信号处理的故障分析方法（80年代末至21世纪初），1-2 段
- 第三阶段：传统机器学习方法（21世纪初至2015年前后），1-2 段
- 第四阶段：深度学习方法（2015年至今），2-3 段
- 每段自然衔接，不加粗阶段标题，以学术综述论文的叙述风格行文
- 每个阶段引用 3-5 篇真实文献

**## 2.2 国内外研究现状与发展趋势（2000-3500 字，7-9 自然段）**

用 7-9 个自然段覆盖以下内容：
- 国外研究现状（3-4 段）：按国家/机构分述（美国、英国、日本等），每段介绍 2-3 个代表性研究及其方法与成果
- 国内研究现状（2-3 段）：按高校/科研院所分述（清华、浙大、华北电力、中国电科院等），每段介绍 2-3 项代表性工作
- 发展趋势（2 段）：总结 5-6 个主要趋势方向
- 全程以自然段行文，不用加粗小标题分隔

**## 2.3 国内外研究机构对本项目的研究情况（1500-2000 字，7-9 自然段）**

用 7-9 个自然段分析现有研究不足并提出本项目创新点：
- 现有研究不足分析（4-5 段）：每段聚焦一个不足方向，结合具体文献指出局限
- 本项目创新点（3-4 段）：每段阐述一项核心创新，说明如何弥补上述不足
- 以自然段行文，不用"不足一""创新一"等编号前缀

**参考文献（60-100 篇）**

**⚠️ 必须联网检索：** 使用 WebSearch 工具在 Google Scholar、IEEE Xplore、CNKI 等数据库检索真实论文，确认作者、刊名、卷期页码准确无误。严禁凭记忆编造文献。

格式遵循 GB/T 7714-2015：
- 中英文混合，英文文献≥30篇，中文文献≥20篇
- 优先使用近 5 年文献（2020年以后≥50%）
- 涵盖期刊论文[J]、会议论文[C]、专著[M]、学位论文[D]等多种类型
- IEEE Transactions on Power Delivery / Smart Grid / Power Systems、IET Generation Transmission & Distribution、Applied Energy、中国电机工程学报、电力系统自动化、电网技术等为主要来源
- 覆盖本章引用的所有文献编号

```
> [1] 作者.题名[J].刊名,年,卷(期):起止页码.
> [2] Author. Title[J]. Journal, Year, Vol(Issue): Pages.
> [3] 作者.题名[D].保存地:保存单位,年.
```

## 写作规范与要求

### 语言风格
- 正式学术语言，避免口语化
- 逻辑严谨，论述层层递进
- 使用"本项目""本研究"等专业表述
- 段落间有过渡衔接，避免突兀跳转
- **⚠️ 以自然段形式行文，严禁使用AI味重的"加粗关键词+冒号+内容"列表风格**
- 除第五章的量化目标和进度表格外，正文应以连贯段落为主
- 每段有明确的中心思想，段落之间有自然过渡

### 深度要求
- **不要泛泛而谈**：每个观点必须有具体的数据、案例、文献支撑
- **不要复制粘贴**：各章节的表述避免重复
- **不要空洞论述**：痛点要具体、目标要量化、方法要详细
- **每段 150-300 字**：段落不宜过短（<100字显单薄）也不宜过长（>400字影响可读性）

### 内容导向
- **国网/南网视角**：始终从电网公司角度出发
- **政策导向**：紧密结合国家能源政策、电力改革政策
- **问题导向**：先讲清楚痛点，再提出解决方案
- **效益导向**：强调经济效益和社会效益
- **创新导向**：突出技术创新点和行业领先性

### 适配领域

可适配国网/南网以下主要技术方向：

| 方向 | 典型主题 |
|------|----------|
| 电网规划与运行 | 新型电力系统规划、调度优化、配电网智能化、输变电技术 |
| 数字化与智能化 | 大数据分析、AI应用、数字孪生、物联网、边缘计算 |
| 新能源与储能 | 新能源并网消纳、储能技术、分布式能源、虚拟电厂 |
| 营销与服务 | 负荷预测、用户画像、需求侧管理、电力市场交易 |
| 安全与可靠性 | 电网安全防护、设备监测、故障诊断预警、网络安全 |
| 绿色低碳 | 碳排放监测、能效管理、电能替代、综合能源服务 |

## 关键注意事项

1. **分轮生成**：每轮只生成 2-3 章，确保深度。不要试图一次生成全部内容
2. **字数门槛**：每轮不少于 2500 字，全文不少于 15000 字
3. **不编造数据**：经费、时间节点、性能指标等使用占位符"XX"
4. **参考文献必须联网检索**：使用 WebSearch 工具检索真实文献，严禁凭记忆编造。总量 60-100 篇
5. **技术准确性**：技术原理描述准确，不夸大、不误导
6. **各章联动**：第三章的关键点难点要与第四章的研究内容呼应；第四章的课题与第五章的目标对应
7. **Markdown 格式**：严格遵循格式约定，确保后续 docx-cn 能正确解析
8. **自然段写作**：正文以自然段形式行文，禁止大量使用加粗前缀列表。学术综述、技术原理等章节尤其需要以连贯段落展开论述
