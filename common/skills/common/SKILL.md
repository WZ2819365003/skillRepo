# 通用文档结构化生成技能 v1.0

## 核心理念

用户提供**参考材料**（示例文档、模板文件）和**目的/主题**，本技能：
1. 解析参考材料的文档结构（标题层级、章节划分、段落模式、表格、公式等）
2. 按照解析出的结构，分块生成新主题的 Markdown 内容
3. 调用 docx-cn 技能将 Markdown 转为格式规范的 Word 文档

```
用户输入：参考文件 + 目的/主题
    ↓
[Phase 1] 解析参考文件结构 → 输出结构蓝图（JSON）
    ↓
[Phase 2] 按结构蓝图分轮生成 Markdown（每轮 2-3 章）
    ↓
[Phase 3] 合并为完整 .md 文件
    ↓
[Phase 4] 调用 docx-cn 技能 → 输出 .docx 文件
```

---

## Phase 1：解析参考文件结构

### 支持的参考文件格式

| 格式 | 读取方式 |
|------|----------|
| `.md` | 直接 Read 工具读取 |
| `.txt` | 直接 Read 工具读取 |
| `.docx` | Read 工具读取（Claude Code 支持） |
| `.pdf` | Read 工具读取（需指定页码范围） |
| 用户直接粘贴文本 | 从对话中提取 |

### 结构解析规则

读取参考文件后，按以下维度分析其结构：

**1. 标题层级体系**
- 识别所有标题层级（一级/二级/三级/四级）
- 记录每个标题的编号风格（如"第一章"/"一、"/"1.1"/"1.1.1"）
- 记录标题层级数量

**2. 章节内容模式**（对每个章节分析）
- 章节主题/功能（如"背景介绍""文献综述""技术方案""进度计划"等）
- 内容类型：纯段落 / 段落+列表 / 段落+表格 / 段落+公式
- 预估字数范围
- 段落风格：自然段叙述 / 加粗前缀列表 / 编号列表
- 是否包含参考文献引用

**3. 特殊元素**
- 表格：位置、列数、用途（数据表/进度表/对比表等）
- 图片/图片占位符：位置、用途
- 公式：是否包含数学公式
- 参考文献：数量、格式风格

**4. 文档元信息**
- 文档类型（报告/论文/方案/标书等）
- 适用的 docx-cn 模板（default/thesis/paper/report）
- 是否需要封面、目录

### 结构蓝图输出格式

解析完成后，在回复中输出结构蓝图（不写文件），格式如下：

```
## 结构蓝图

**文档类型：** [类型]
**docx-cn 模板：** [default/thesis/paper/report]
**封面：** [是/否] → [封面信息字段]
**目录：** [是/否]
**总预估字数：** [X-Y 字]
**生成轮数：** [N 轮]

### 章节结构

| 轮次 | 章节 | 层级 | 内容功能 | 内容类型 | 预估字数 | 特殊元素 |
|------|------|------|----------|----------|----------|----------|
| 1 | 第一章 XX | H1 | 背景与意义 | 自然段 | 3000-4000 | 无 |
| 1 | 1.1 XX | H2 | 研究背景 | 自然段 | 1500-2000 | 无 |
| ... | ... | ... | ... | ... | ... | ... |

### 写作风格要求
- 段落风格：[自然段叙述 / 加粗前缀 / 混合]
- 人称视角：[本项目/本研究/第三人称]
- 专业程度：[学术/工程/通用]
- 参考文献：[需要/不需要]，[数量范围]，[格式标准]
```

输出蓝图后等待用户确认，用户说"继续"或"可以"后进入 Phase 2。如果用户要求调整（如增减章节、改变深度），先修改蓝图再继续。

## Phase 2：分轮生成 Markdown

### 分轮策略

- 总字数 ≤ 5000 字：1 轮生成
- 总字数 5000-12000 字：2 轮生成
- 总字数 12000-20000 字：3-4 轮生成
- 总字数 > 20000 字：4-6 轮生成
- 每轮生成 2-3 个章节，确保每章有足够深度
- 每轮结束后将内容写入 `out/{文件名}_round{N}.md`

### Markdown 格式约定

与 docx-cn 的 Markdown 转 Word 映射表一致：

```markdown
# 一级标题（H1，前分页）
## 二级标题（H2）
### 三级标题（H3）
#### 四级标题（H4）

正文段落直接书写，每段之间空一行。首行缩进由 Word 生成时自动添加。

**加粗前缀：**后接普通正文，用于效益描述、要点强调等。

（1）编号列表项。

（2）第二项。

$$E = mc^2$$

行内公式 $\alpha + \beta = \gamma$ 嵌入正文。

> [1] 参考文献条目，GB/T 7714-2015 格式。

![图X.X 图片标题](描述：详细描述图片内容)

| 列A | 列B | 列C |
|-----|-----|-----|
| 值1 | 值2 | 值3 |
```

### 写作核心原则

1. **结构来自参考材料，内容来自用户主题** — 模仿参考材料的章节划分、段落模式和深度层次，但内容完全围绕用户指定的新主题
2. **自然段优先** — 除非参考材料明确使用列表风格，否则默认以自然段叙述。禁止无脑使用"加粗关键词+冒号+内容"的 AI 列表风格
3. **深度匹配** — 每章的字数和深度应与参考材料中对应章节相当。参考材料中 2000 字的章节，生成时也应达到 1500-2500 字
4. **专业术语一致** — 保持与用户主题领域一致的专业术语和表述习惯
5. **不抄袭参考内容** — 参考材料仅供结构和风格参考，不复制其具体内容、数据、案例

### 每轮生成的交互流程

```
[生成 Round N 内容]
    ↓
写入 out/{name}_round{N}.md
    ↓
提示用户："Round N 已完成（第X章-第Y章，约Z字）。继续生成下一轮？"
    ↓
用户确认后继续 Round N+1
```

如果用户指定"一口气生成完"或"不用确认"，则连续生成所有轮次。

---

## Phase 3：合并 Markdown

所有轮次完成后，合并为一个完整的 `.md` 文件：

```
out/{主题简写}_full.md
```

合并时：
- 保留第一轮的封面/frontmatter 信息
- 更新目录章节列表
- 去除每轮的轮次标记（如"第X轮内容"）
- 统一参考文献编号（如果多轮都有参考文献，合并编号）
- 检查章节之间的过渡是否自然

---

## Phase 4：调用 docx-cn 转 Word

将合并后的 `.md` 文件交给 docx-cn 技能处理：

1. 根据结构蓝图中确定的模板类型，编写 MD → docx 转换脚本
2. 脚本必须实现 docx-cn SKILL.md 中"Markdown 转 Word 工作流"的所有元素映射，包括：
   - **LaTeX 公式**：`$$块级公式$$` → 居中 Math 段落，`$行内公式$` → DocxMath 内嵌
   - **表格**：首行为表头（蓝底加粗），`tableHeader: true` 跨页重复
   - **图片占位符**：灰色框 + 图注
   - **参考文献**：悬挂缩进
   - **加粗前缀段落**：`bodyParagraphWithBoldPrefix()`
3. 执行脚本生成 `.docx` 文件
4. 提示用户打开后按 Ctrl+A → F9 刷新目录

### 输出文件

```
out/{主题简写}_full.md          — 完整 Markdown 源文件
out/{主题简写}.docx             — 最终 Word 文档
out/md2docx_{主题简写}.js       — 转换脚本（可复用）
```

## 完整工作流示例

**用户输入：**
> 请参考 target/示例报告.docx，帮我生成一个关于"智慧城市交通信号优化系统"的可行性研究报告的 Word

**执行流程：**

1. Read `target/示例报告.docx`，分析其结构
2. 输出结构蓝图，等待用户确认
3. 用户确认后，分 3 轮生成 Markdown：
   - Round 1：封面 + 第一章（目的和意义）
   - Round 2：第二章（国内外研究综述）
   - Round 3：第三章（技术方案）+ 第四章（实施计划）
4. 合并为 `out/智慧交通信号优化_full.md`
5. 编写转换脚本，生成 `out/智慧交通信号优化.docx`

## 特殊场景处理

### 用户只给主题没给参考文件

如果用户只说"帮我写一个xxx的word"但没有提供参考文件：
- 询问用户是否有参考文件
- 如果没有，根据文档类型使用内置默认结构模板：
  - **可研报告** → 8 章标准结构（目的意义/综述/理论依据/研究内容/预期目标/成果转化/进度/组织管理）
  - **技术方案** → 5 章结构（项目概述/需求分析/技术方案/实施计划/预算）
  - **学术论文** → 引言/相关工作/方法/实验/结论
  - **通用报告** → 3-5 章自由结构

### 参考文件很长

如果参考文件超过 500 行，Phase 1 只需要阅读其标题结构和前 2-3 个章节的段落模式，不必逐行精读全文。关键是提取**结构蓝图**，而非理解全部内容。

### 用户要求"完全照搬格式"

如果用户明确要求严格遵循参考文件的格式（包括标题编号风格、章节数量、段落数量等），则在结构蓝图中标注"严格模式"，生成时完全复刻参考材料的章节数和每章段落数。

### 多个参考文件

如果用户提供多个参考文件，取各文件结构的并集/最优结构，并在蓝图中注明参考了哪些文件的哪些部分。

---

## 关键注意事项

1. **参考结构不参考内容** — 参考材料仅供提取文档骨架，新文档的所有内容必须围绕用户主题原创
2. **分轮生成确保深度** — 不要一次性生成全部内容。每轮 2-3 章，每章保证字数达标
3. **自然段写作** — 默认以自然段叙述，避免 AI 味重的列表化写作
4. **公式必须用 Math 对象** — Markdown 中的 `$...$` 和 `$$...$$` 在转 Word 时必须转为 DocxMath，不能作为纯文本
5. **中文弯引号转义** — JS 脚本中的 `""` 必须写成 `\u201c`/`\u201d`
6. **参考文献联网检索** — 如果需要参考文献，使用 WebSearch 检索真实论文，不凭记忆编造
7. **等用户确认再继续** — Phase 1 输出蓝图后必须等用户确认；每轮生成后提示进度。用户说"一口气生成"则连续执行
